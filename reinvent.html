EKS Auto (Re:Invent 2024)

Amazon EKS Auto Mode was announced at Re:Invent 2024 to deliver a fully managed Kubernetes experience:

  • Automatic Compute — No node groups to configure. EKS Auto provisions the right EC2 instances (powered by Karpenter) based on pod requirements. Supports Spot, On-Demand, and Graviton automatically.
  • Automatic Networking — VPC CNI, CoreDNS, and kube-proxy are managed and upgraded automatically. Load balancer integration is built in.
  • Automatic Storage — EBS CSI driver is pre-installed and managed. PersistentVolumeClaims just work.
  • Automatic Upgrades — Kubernetes version, AMIs, and add-ons are upgraded automatically with minimal disruption.

Impact: EKS Auto eliminates the undifferentiated heavy lifting of managing Kubernetes infrastructure, letting teams focus purely on deploying and running applications.

Advanced

Aurora DSQL (Re:Invent 2024)

Amazon Aurora DSQL (Distributed SQL) was announced at Re:Invent 2024 as a serverless, distributed SQL database:

  • Distributed & Multi-Region — Active-active writes across multiple AWS regions with strong consistency. No read replicas needed — every region can serve reads and writes.
  • PostgreSQL Compatible — Standard PostgreSQL interface, so existing applications and tools work without modification.
  • Serverless — No capacity planning, no instances to manage. Scales automatically based on workload demand.
  • Strong Consistency — Unlike DynamoDB Global Tables (eventual consistency), Aurora DSQL provides serializable isolation across regions.

Use cases: Global applications requiring consistent reads/writes across regions, financial systems needing ACID transactions globally, replacing complex multi-region database replication setups.

Intermediate

S3 Tables (Re:Invent 2024)

Amazon S3 Tables, announced at Re:Invent 2024, brings native Apache Iceberg table support to S3:

  • Structured Data in S3 — Store tabular data (rows and columns) directly in S3 with built-in table format support. No need to manage a separate metastore.
  • Apache Iceberg Format — Uses the open-source Iceberg table format natively. Supports schema evolution, time travel, partition evolution, and ACID transactions.
  • Query with Standard Tools — Query S3 Tables directly from Amazon Athena, EMR, Redshift, and third-party tools that support Iceberg.
  • Automatic Optimization — S3 manages compaction, snapshot management, and unreferenced file cleanup automatically.

Impact: S3 Tables simplifies the data lakehouse architecture by eliminating the need for separate Glue Catalog, manual Iceberg table maintenance, and complex compaction jobs. It's S3 evolving from an object store into a full-featured data platform.
